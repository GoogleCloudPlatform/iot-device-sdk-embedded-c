181 /* @union iotc_crypto_key_union_t
182  * @brief A union which describes the private key data which will be passed to
183  * the crypto BSP to handle JWT signatures. There are three options:
184  *
185  * 1. key: pass a byte array to the PEM formatted private key.
186  * 2. key_slot: used for multi-slotted secure elements which can store
187  * private_keys in secure storage and use them to sign data.
188  * 3. key_custom: a catch all for other options where the client application
189  * may pass untyped data to the BSP crypto implementation.
190  */
